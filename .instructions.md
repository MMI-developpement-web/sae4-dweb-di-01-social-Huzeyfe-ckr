# 📋 SAE4 DWeb DI 01 - Social Network Project

## 🎯 Aperçu du Projet
Application de réseau social moderne construite avec React + TypeScript (frontend) et Symfony 7.4 + Doctrine/MySQL (backend).
- **Frontend**: Vite, React 18, TypeScript, Tailwind CSS, React Router v6
- **Backend**: Symfony 7.4, Doctrine ORM, MySQL 8
- **Design**: Figma MCP (Model Context Protocol) - respecter strictement les designs fournis
- **Base de données**: MySQL via Docker, gérable via phpMyAdmin (http://localhost:8070)

## 🧩 Architecture des Composants UI

### Pattern CVA + Interface

Tous les composants UI suivent ce pattern :

```tsx
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "../../lib/utils.ts";

// 1. Définir variants avec CVA
export const componentVariants = cva("base-classes", {
  variants: {
    variant: {
      solid: "...",
      dark: "...",
    },
    size: {
      sm: "...",
      md: "...",
    },
  },
  defaultVariants: {
    variant: "solid",
    size: "md",
  },
});

// 2. Typer les props avec Interface + VariantProps
interface ComponentProps
  extends VariantProps<typeof componentVariants> {
  children?: React.ReactNode;
  icon?: React.ReactNode;
  className?: string;
  // autres props...
}

// 3. Composant avec cn() pour fusionner classes
export default function Component({ 
  children, 
  variant, 
  size, 
  className,
  ...rest 
}: ComponentProps) {
  const classes = cn(componentVariants({ variant, size }), className);
  return <div className={classes}>{children}</div>;
}
```

### Composants Existants

#### Input
- **Variants**: `default`, `subtle`, `light`, `textarea`
- **Props**: `type` (text/email/password), `placeholder`, `value`, `onChange`, `as` (input/textarea), `showToggle` (password eye toggle)
- **Usage**: Tous les formulaires

#### Button
- **Variants**: `solid`, `dark`, `icon`, `icons`, `post`
- **Props**: `icon`, `iconPosition` (left/right), + tous les `ButtonHTMLAttributes` (type="submit", disabled, etc.)
- **Usage**: Tous les formulaires, actions

#### Avatar
- **Variants**: `mehmet`, `leo`, `mao`, `default`
- **Sizes**: `sm`, `md`, `lg`, `xl`
- **Usage**: Profils utilisateurs, posts

#### Header/Footer
- **Header**: Logo + navigation
- **Footer**: Boutons navigation (Home, Search, Notifications, Messages)

## 📝 Conventions Coding

### TypeScript
- Tous les composants doivent avoir des interfaces explicites
- Typage strict: pas de `any`
- Props interfaces plutôt que `React.FC<Props>`

### Nommage
- Fichiers composants: PascalCase (Button.tsx)
- Fichiers routes: lowercase (login.tsx)
- Variables/fonctions: camelCase
- Constantes: UPPER_SNAKE_CASE

### Imports
- Imports relatifs relatifs depuis la racine `src/`
- Organiser: React/libs d'abord, puis composants locaux

```tsx
// ✅ BON
import { useNavigate } from 'react-router-dom'
import Input from './ui/Input'
import Button from './ui/Button'

// ❌ MAUVAIS
import React, { Component } from 'react'
import Input from '../../../ui/Input'
```
