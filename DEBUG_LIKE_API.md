# Debug Instructions for Like API 401 Issue

## Quick Diagnosis

**Problem:** Frontend POST /api/posts/{id}/like returns 401 Unauthorized, but backend accepts the same token via curl.

**Root Cause:** Unknown - token appears to be stored or transmitted incorrectly by frontend.

## What We Know ✅

1. **Backend is WORKING**: `curl login` + `curl post /like` = SUCCESS
2. **Fresh tokens work**: Tokens generated by login work immediately 
3. **CORS is configured correctly**: Preflight returns proper headers
4. **API format is correct**: Response structure matches expectations

## What's Broken ❌

- Frontend receives 401 when attempting POST like
- Token might not be stored in localStorage correctly
- OR token might be corrupted during transmission
- OR token might expire too quickly

## How to Debug (Step by Step)

### Method 1: Use the Debug Tool (Easiest)

1. Open your browser
2. Go to: `http://localhost:5173/public/debug-like-API.html`
   - (Or wherever frontend is hosted)
3. Click buttons in order:
   - **Step 1**: Calls login, stores token in localStorage
   - **Step 2**: Checks if token is actually stored
   - **Step 3**: Sends like request with stored token
   - **Full Flow**: Runs all 3 steps automatically
4. Check the colored status boxes for results:
   - 🟢 Green = Problem is likely elsewhere
   - 🔴 Red = Found the issue
   - 🟡 Yellow = Inconclusive

### Method 2: Manual Console Debugging

1. Open app at http://localhost:5173 (or nginx address)
2. Press **F12** to open Developer Tools
3. Go to **Console** tab
4. Click **Login** and enter:
   - Username: `testuser1774262871`
   - Password: `testpass123`
5. Watch console for logs starting with:
   - `=== LOGIN RESPONSE ===`
   - `✅ Token stored in localStorage`
   - `📌 Token from getAuthToken(): ...`
6. After login, go to **Home** page
7. Click like button on any post
8. In console, look for logs from `likePost()` function showing:
   - Token presence/absence
   - Headers sent
   - Response status
9. Check the **Network** tab (in DevTools) for the actual request:
   - Look for POST /api/posts/{id}/like request
   - Click it and check **Request Headers**
   - Verify `Authorization: Bearer {token}` is present

### Method 3: Storage Inspector

In browser console, run:
```javascript
// Check if token is stored
console.log('authToken:', localStorage.getItem('authToken'));
console.log('currentUser:', localStorage.getItem('currentUser'));

// Check if it's the right length (should be 64 hex chars)
const token = localStorage.getItem('authToken');
console.log('Token length:', token?.length);
console.log('Is hex?', /^[a-f0-9]{64}$/.test(token || ''));
```

## What to Look For

### ✅ Signs Everything is Working:
- Login console shows: `✅ Token stored in localStorage`
- `localStorage.getItem('authToken')` returns a 64-character hex string
- Network tab shows `Authorization: Bearer {token}` header
- No "Invalid token" or "Unauthorized" errors
- Like button toggles and UI updates

### ❌ Signs of Problems:
- `localStorage.getItem('authToken')` returns `null`
- Token length is not 64 characters
- Authorization header is missing in Network tab
- Console shows `❌ TOKEN MISMATCH!`
- Backend log shows `"Invalid token"` error
- Token changes between login and like request

## If Debug Tool Shows 🔴 Red Status:

### If Step 1 fails (Login):
- Backend login endpoint is broken
- Check: `curl -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d '{"user":"testuser1774262871","password":"testpass123"}'`

### If Step 2 fails (Storage):
- localStorage is disabled or full
- Browser might be in incognito mode (localStorage disabled in some browsers)
- Try: Open DevTools Console and run `localStorage.setItem('test', 'value')`

### If Step 3 fails with 401 (Like):
- **Most likely**: Token isn't stored correctly, or expires immediately
- Try: Clear localStorage and login again
- Try: Different browser (might be extension blocking locals storage)
- Check: Backend `access_token` table to see if tokens expire too quickly

### If Step 3 fails with 409 (Already liked):
- ✅ This is GOOD! It means token was accepted!
- Problem is not authentication, but application logic
- Unlike the post first, then try again

## Commands to Verify Backend Works

```bash
# Generate fresh token
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"user":"testuser1774262871","password":"testpass123"}' | jq -r '.token')

# Test like endpoint with that token
curl -X POST http://localhost:8080/api/posts/50/like \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" | jq .

# Should return either:
# - {"message":"Post liké","likes":1}  [201]
# - {"error":"Vous avez déjà liké ce post"}  [409]
# - NOT: 401 Unauthorized
```
